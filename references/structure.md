## 一、全量结论

写作框架的核心不是复杂，而是让读者从“被标题击中”一路走到“认同观点”。全量复检发现，最常见的结构是：痛点开场、故事/例子推进、观点升级、金句收束。不同文章会变化成清单式、故事式、热点评论式、情绪独白式等。

## 三、最常见的推进路径

1. 开头先给痛点或冲突，不慢热。
2. 中段用故事和例子证明“这不是个例”。
3. 再用反常识观点把故事抬高。
4. 最后用一句强判断收束，让读者能转发。

## 六、不同类型文章的框架选择

关系类文章：适合“冲突场景 -> 双方误区 -> 真实需求 -> 边界判断”。

职场类文章：适合“普遍焦虑 -> 现实案例 -> 反鸡汤观点 -> 可执行建议”。

金钱类文章：适合“争议判断 -> 生活例子 -> 价值解释 -> 行动提醒”。

亲情类文章：适合“具体场景 -> 情绪唤醒 -> 愧疚/理解 -> 温柔收束”。

社会议题：适合“事件 -> 普通人处境 -> 底线判断 -> 价值立场”。

## 七、框架自检标准

1. 开头 3 段内是否出现明确冲突？
2. 中段是否有至少 2 个具体例子？
3. 观点是否有递进，而不是重复标题？
4. 结尾是否有可转述的句子？
5. 每一部分是否都服务同一个主观点？

## 十、框架选择的具体判断

如果文章主题是恋爱、婚姻、婆媳、朋友关系，优先用关系冲突框架：先写一个让人不舒服的小场景，再追问这个不舒服背后的权力、尊重、边界问题。

如果文章主题是职场、金钱、个人成长，优先用现实压力框架：先写普通人的焦虑，再指出鸡汤答案为什么失效，最后给一个更现实的判断。

如果文章主题是母爱、父爱、亲子育儿，优先用情绪递进框架：先写具体生活细节，再写亏欠、理解或心疼，最后落到陪伴和责任。

如果文章主题是社会议题，优先用公共底线框架：先让读者看到事件里的受害者，再把愤怒落到规则、尊严和安全感上。

## 十一、框架拆解模板补充

31. 开头：用一句反常识判断打破读者默认想法。
32. 开头：用一个具体关系冲突替代抽象议题。
33. 开头：用读者最熟悉的生活细节进入。
34. 中段：先放故事，再放判断，不要一上来讲大道理。
35. 中段：每个例子后面都要回扣主观点。
36. 中段：至少安排一次“你以为 A，其实 B”的观点翻转。
37. 中段：如果情绪很重，要补一个现实解释。
38. 中段：如果道理很重，要补一个生活例子。
39. 结尾：不要复述标题，要升级标题。
40. 结尾：用一句能截图的判断替代普通总结。
41. 复检：删掉和主观点无关的例子。
42. 复检：检查每一段是否都在推进同一个问题。
43. 复检：检查故事是否有冲突、代价、转折。
44. 复检：检查观点是否能被读者复述。
45. 复检：检查结尾是否给了情绪出口。

## 十二、框架复检后的最终判断

写作框架不是固定套壳，而是服务于读者情绪推进。1012 篇全量复检后，最稳定的框架就是：先击中，再证明，再升级，再收束。
